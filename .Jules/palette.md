## 2026-07-24 - [Form and Image Upload Accessibility Improvement]
**Learning:** Standard interactive HTML forms often miss proper label-to-input associations, causing poor screen reader accessibility. Interactive file uploads with custom visual containers (like avatar/profile photo preview) must use Tailwind's `sr-only` instead of hiding the input entirely with `hidden`. This ensures the input remains focusable in the tab sequence. Focus rings and overlay styling should be managed dynamically using `focus-within:` on parent containers and `group-focus-within:` or `focus-within:` classes on child overlay labels.
**Action:** Always replace `hidden` with `sr-only` for inputs to keep them in focus/tab order, use `focus-within:` classes on parent container components, and add matching `for` attributes to labels to establish solid relationships for assistive technologies.
