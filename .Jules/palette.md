## 2026-03-31 - Enhancing File Upload and Form Controls Keyboard Accessibility
**Learning:** File upload controls built using `<input type="file" class="hidden">` are completely invisible to keyboard-only and screen reader navigation. Replacing `class="hidden"` with `class="sr-only"` keeps the input visually hidden while ensuring it remains focusable during keyboard tab navigation. Associating focus indicators using Tailwind's `focus-within:` on the visual container or label allows keyboard users to see where they are focused on.
**Action:** Always prefer `sr-only` instead of `hidden` or `display: none` for inputs that need to trigger file uploading, and use `focus-within:` on wrapping labels or container elements to provide feedback. Ensure form labels are explicitly linked via `for` attributes to inputs, and icon-only buttons feature explicit `aria-label` labels.
