## 2026-07-23 - [Keyboard Accessibility for Avatar Upload and Icon-Only Buttons]
**Learning:** Hidden inputs (`display: none` or Tailwind's `hidden`) cannot receive focus from keyboard users, making file uploads impossible via keyboard navigation. Replacing `hidden` with Tailwind's `sr-only` keeps the input accessible in the DOM for keyboard and screen-readers. Containers of such inputs must utilize `focus-within:` classes to trigger focus rings, and labels must use `group-focus-within:` to show overlay text on focus. Additionally, icon-only buttons require `aria-label` to provide textual descriptions for screen readers.
**Action:** Always use `sr-only` instead of `hidden` for interactive elements that need keyboard access, leverage `focus-within` for visual indicators on container borders/focus states, and add explicit `aria-label` to all icon-only buttons.
